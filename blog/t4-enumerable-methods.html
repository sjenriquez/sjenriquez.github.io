Today's enumerable is:

#map

11/1/14

Today we'll look at the #map method in the Enumerable module. To break that down a bit, Ruby has various modules that contain different methods. Classes, like Array and String, can use various modules with all their associated methods.

The #map method is very useful. It will run a block of code on every element you throw at it. Here's the syntax:

map { |obj| block } --magically-returns--> array


When you call #map, Ruby will pass each element through the block of code, #map has a spaceholder variable you name (here it's "obj") to temporarily store the current element while the block of code is run on that element. This repeats for each element and then Ruby outputs the new array. Let's say we have an array of states with all lowercase letters and we want to capitalize each state. Here's how we do it:

states = ["maryland", "virginia", "maine"]

states.map { |current_state| current_state.capitalize }

=> ["Maryland", "Virginia", "Maine"]


Snazzy. It's important to remember that #map will not alter the array passed in, if you DO want to change it (referred to as "destructive"), use #map!. Otherwise, you can store the output in a new variable:

states = ["maryland", "virginia", "maine"]

cap_states = states.map { |current_state| current_state.capitalize }

puts cap_states

=> ["Maryland", "Virginia", "Maine"]


This, of course, is only the tip of the iceberg, #map can be combined with other methods to create much more complex code. Try something a little more complex yourself, what happens if our state array is:

states = ["maryland", "virginia", "maine", "new york", "new jersey"]


Give it a try!
