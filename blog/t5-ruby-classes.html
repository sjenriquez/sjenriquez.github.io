Ruby Classes

Let's take the motorcycle out for a ride

11/9/14

Today we'll construct a ruby class for our motorcycle. You can think of the class as a way of creating our motorcycle and class methods as actions we want our motorcycle to do. Let's get started by setting up our motorcycle class.

class Motorcycle

def initialize @motorcycle = "" end end


The code above allows us to create a new class Motorcycle and initializes a new instance variable @motorcycle. Instance variables can be referenced and accessed by class methods. Let's create a few methods for our motorcycle.

class Motorcycle

   def initialize
     @motorcycle = ""
   end
   def brand
     puts "Please enter your motorcycle brand:"
     @motorcycle = gets.chomp
   end

   def rev
     puts "My #{@motorcycle} likes to go, vroom vroooommm..."
   end

   def my_ride
     puts "I ride a #{@motorcycle} and it's fast!"
   end
end

my_bike = Motorcycle.new
my_bike.brand
my_bike.rev
my_bike.my_ride


Now we have a few methods we can call on our new motorcycle. Let's take a look at how that would ride out. The program first asks us what brand of motorcycle we want.

$ Please enter your motorcycle brand:
Moto Guzzi

My personal ride is a Moto Guzzi. After I enter my brand a few methods run, here's what we'll see.

My Moto Guzzi likes to go, vroom vroooommm...
I ride a Moto Guzzi and it's fast!


As you can see, classes contain methods that act upon new instances of our class. May you always code and ride fast!
