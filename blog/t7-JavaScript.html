Ruby vs. JavaScript

Hashes and Objects

11/21/14

Ruby and JavaScript have some distinct differences. Both are Object Oriented languages and therefor have similarities as well. This blog post will compare and contrast Ruby Hashes, JavaScript Objects, and give a few examples of use.


Ruby-doc.org defines as Hash as:

A Hash is a dictionary-like collection of unique keys and their values. Also called associative arrays, they are similar to Arrays, but where an Array uses integers as its index, a Hash allows you to use any object type.


Hashes give us one more definable element than an array. We can iterate through them with methods like #each and #map or access single keys and values. Hashes, like everything in Ruby, are objects belonging to the Hash class. They are a very useful data structure, particularly for dictionaries.


Here's an example of a #hash in ruby representing a band.

1 beatles = {
2   "paul" => "bass",
3   "john" => "guitar",
4   "george" => "guitar",
5   "ringo" => "drums",
6   num_of_albums: 13,
7   "debate about # albums" => true
8 }


As you can see, both the keys and values can consist of different objects, including #arrays.


JavaScript objects may at first appear to be like a Ruby hash. An object in JavaScript also has multiple values stored in pairs.

1 var beatles = {
2   paul: "bass",
3   john: "guitar",
4   george: "guitar",
5   ringo: "drums",
6   numOfAlbums: 13,
7   debateAboutNumOfAlbums: true
8 }


These pairs are referred to as property and value, numOfAlbums: 13. Naming conventions are slightly different in JavaScript as you can see. Unlike Ruby hashes, we can't use any value type as the property. Properties are just properties. Also unlike Ruby, in JavaScript we can make a property's value a method. For example:

1 var beatles = {
2   paul: "bass",
3   john: "guitar",
4   george: "guitar",
5   ringo: "drums",
6   numOfAlbums: 13,
7   debateAboutNumOfAlbums: true
8   ringosInstrument: function() {
9     return "Ringo plays" + this.ringo;
10  }
11 }
12
13 console.log(beatles.ringosInstrument())

Ringo plays drums


Here we see a key difference. We can run various methods on a Ruby #hash but methods are defined inside a Class, not in the #hash object itself. Let's put this hash inside a Class so we can define a method to print what Ringo plays.


Another difference appears between JavaScript and Ruby when we head down this road. In Ruby there is strict encapsulation of variables. Variables within Classes aren't accessible unless we explicitly define them as so. Furthermore, variables are accessible within a class only when defined as an instance variable. JavaScript variables are globally accessible when we define them as var someVar = something. JavaScript doesn't have strict encapsulation like Ruby. Let's finish out this post showing how to print "Ringo plays drums" in Ruby.

1 class Best_band
2
3   def initialize
4     @beatles = {             # Create instance variable
5         "paul" => "bass",
6         "john" => "guitar",
7         "george" => "guitar",
8         "ringo" => "drums",
9         num_of_albums: 13,
10        "debate about albums" => true
11        }
12  end
13
14  def ringo_plays
15    puts "Ringo plays #{@beatles["ringo"]}"
16  end
17
18 end
19
20 new_object = Best_band.new
21
22 new_object.ringo_plays

Ringo plays drums

Ruby is very beginner friendly and makes wrapping your head around Object Oriented Programming very easy. If you start with Ruby, JavaScript might be confusing at first. Keep at it!
